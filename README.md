# SwiftMT103Parser

Уеб API, изградено с **.NET 10**, което парсира SWIFT MT103 съобщения от текстови файлове, съхранява данните в SQLite база данни и ги връща като JSON.

---

## Какво е SWIFT MT103?

MT103 е стандартен SWIFT формат за международни парични преводи между банки. Всяко съобщение съдържа структурирани блокове с информация за наредителя, получателя, сумата, валутата и допълнителни детайли по транзакцията.

## Парсирани полета

Приложението разпознава и разделя всички блокове и полета на MT103 съобщението:

### Блок 1 — Application Header
| Поле | Описание |
|---|---|
| ApplicationId | Идентификатор на приложението (F) |
| ServiceId | Тип услуга (01) |
| LogicalTerminal | BIC адрес на изпращача |
| SessionNumber | Номер на сесия |
| SequenceNumber | Поредност |

### Блок 2 — Basic Header
| Поле | Описание |
|---|---|
| Direction | I (Input) или O (Output) |
| MessageType | Тип съобщение (103) |
| ReceiverAddress | BIC на получаващата банка |
| Priority | Приоритет (N/S/U) |

### Блок 3 — User Header
| Поле | Описание |
|---|---|
| ValidationFlag | Флаг за валидация (напр. STP) |

### Блок 4 — Text Block (основни полета)
| Поле | Описание |
|---|---|
| :20 | Референтен номер на транзакцията |
| :23B | Код на банковата операция |
| :32A | Дата на вальор + Валута + Сума |
| :33B | Валута + Сума на нареждането |
| :50K | IBAN + Име + Адрес на наредителя |
| :57A | Банка на получателя (BIC) |
| :59 | IBAN + Име + Адрес на получателя |
| :70 | Основание за превода |
| :71A | Код за такси (SHA / OUR / BEN) |

### Блок 5 — Trailer
| Поле | Описание |
|---|---|
| MAC | Код за автентичност |
| CHK | Контролна сума |

## Как се стартира

### Изисквания
- .NET 10 SDK

### Стъпки

```bash
# 1. Клонирай репото
git clone https://github.com/Kokata23/swift-mt103-parser.git
cd swift-mt103-parser

# 2. Стартирай приложението
dotnet run
```

Приложението стартира на `http://localhost:5000`.  
Swagger UI се отваря директно в браузъра на същия адрес.

## API Ендпойнти

### POST `/api/swift/upload`
Качва `.txt` файл с MT103 съобщение, парсира го и го записва в базата данни.

**Заявка:**
- Тип: `multipart/form-data`
- Поле: `file` — `.txt` файл с MT103 съобщение

**Отговор:** JSON обект с всички парснати полета и присвоено `id`.

---

### GET `/api/swift/messages`
Връща всички записани MT103 съобщения от базата данни.

**Отговор:** JSON масив с всички съобщения, наредени от най-новото.

## Тестване с примерното съобщение

В корена на проекта има файл `sample_mt103.txt` с реално MT103 съобщение.

**Чрез Swagger UI:**
1. Отвори `http://localhost:5000`
2. Избери `POST /api/swift/upload`
3. Натисни „Try it out"
4. Качи `sample_mt103.txt`
5. Натисни „Execute"

**Чрез curl:**
```bash
curl -X POST http://localhost:5000/api/swift/upload \
  -F "file=@sample_mt103.txt"
```

## Логване

Логовете се записват едновременно в конзолата и в папка `logs/`:

```
logs/swift-YYYY-MM-DD.log
```

Логват се: стартиране на приложението, всяко качване и парсиране на файл, записване в базата данни и грешки.
